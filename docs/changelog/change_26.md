# change_26

## Задача

Превью новости на сайте брать из медиа в Telegram (пересланное фото / GIF-анимация / превью видео / картинка как документ), а не только из заглушек.

## Что сделано

- **`events_covers_root()`** — `storage/events/covers/`.
- **`save_event_cover_from_telegram_message`**: скачивает через Bot API во временный буфер (лимит 4 МБ), пишет `events/covers/{event_id}.{jpg|png|webp|gif}`, **`repo.update_event_cover_path`**.
- Приоритет: **`photo`** (самый большой) → **`animation.thumbnail`** → тело **`animation`**, если mime похож на GIF → **`video.thumbnail`** → **`document`**, если image/\* или расширение картинки.
- **`ml_forward`** (одиночное сообщение и альбом): после успешного `try_ingest_forward_to_site` вызывается сохранение обложки; в альбоме берётся **первое** сообщение с подходящим медиа.
- **`handle_event_message`**: опциональный **`source_message`** — обложка и для анонса не только пересылкой.
- Логи: **`event_cover:`** успех / причина пропуска.

## Ограничения

- Анимация как **MP4 без превью** в Telegram не кладётся в `<img>` на ленте — такие случаи пропускаем (нужен был бы `<video>` или постер).
- В логах всё ещё смотреть **409 Conflict**, если два процесса крутят одного бота.
