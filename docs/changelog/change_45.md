# change_45 — защита `/api/auth/verify` от перебора кода

## Что добавлено

- Минимум **3 секунды** между попытками проверки кода для пары IP + Telegram ID.
- После **3 подряд неудачных** попыток (неверный/просроченный код, нет доступа, неверный формат кода) — **блокировка на 5 минут** для той же пары ключей.
- Успешный вход сбрасывает счётчик ошибок и блокировку.

## Зачем

Без ограничений можно было слать запросы на перебор 6-значного OTP; теперь скорость перебора жёстко ограничена, а серия ошибок быстро уводит в таймаут.

## Почему так

- Тот же подход, что уже есть для `/api/auth/request` (`_auth_rl`): in-memory в процессе uvicorn, без миграций и лишней сложности.
- Ключ `IP:telegram_user_id`: отдельные лимиты для разных аккаунтов с одного NAT.

## Что улучшить позже

- Вынести длительность блокировки и интервал в `config`/env при необходимости.
- При нескольких воркерах — общий лимитер (Redis и т.п.), иначе лимиты действуют только внутри одного процесса.
