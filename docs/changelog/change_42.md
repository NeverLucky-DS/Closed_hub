## Что сделано

- В `/library` больше не подставляется автоматически первый файл темы: превью только если в URL есть `file=…`. На телефоне при «Назад к списку» снова один экран — список, без дубля списка и просмотра.
- На узком экране при открытом файле список скрыт (`hidden lg:block`), блок просмотра на всю ширину; на `lg+` снова две колонки.
- Пустое состояние разделено: «нет файлов» vs «есть файлы, ничего не выбрано» (подсказка на десктопе).
- Закрепление файла за компанией: форма для автора файла **или** веб-админа; если компаний в базе нет — текст со ссылками `/companies/new` и `/companies`.
- POST `/library/attach-company`: явная проверка «файл не найден» вместо смешения с `forbidden`.

## Зачем

- Убрать «поехавшую» вёрстку в PWA и ожидаемое поведение «список / просмотр».
- Дать выбор компании там, где раньше не показывалось (админ или пустой справочник компаний).

## Почему так

- Автовыбор первого файла ломал сценарий «только список» без отдельного флага в URL; явный `file=` проще и предсказуемее.
- Права админа на attach согласованы с остальными админ-действиями в библиотеке.
