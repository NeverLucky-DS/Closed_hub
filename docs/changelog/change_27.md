# change_27

## Что сделано

- В `try_ingest_forward_to_site` убран вызов `llm.dedup_event` для пересылок из бота в сайт.
- Вместо него — проверка **точного дубликата**: нормализация пробелов (`strip` + схлопывание `\s+` в один пробел) и сравнение с последними ~80 `raw_text` в БД.
- Заголовок для БД при пересылке — только `_fallback_event_title` (первая строка после маркера пересылки); нормализованный заголовок от Mistral для этого пути больше не запрашивается.

## Зачем

В логах было `hub_site_sync: dedup skip ... dup=True should_add=False` при **разных** постах (Young Con, Data Dojo): семантический дедуп резал всё как «уже есть», из-за чего `site_event_id=None` и лента пустела.

## Почему так

- Для пересылок канала важнее не терять события, чем отфильтровать «похожие по смыслу» тексты.
- Повторная пересылка **того же** текста по-прежнему отсекается точным совпадением.
- Ручные анонсы в `handle_event_message` по-прежнему используют LLM-дедуп.

## Что можно улучшить

- При желании — отдельный лёгкий промпт только на `normalized_title` для пересылок без решения «дубликат/нет» (сейчас экономим вызов Mistral на этом шаге).
