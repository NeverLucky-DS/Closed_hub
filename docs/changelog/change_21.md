# change_21

## Что сделано

- **Страница участника (`person.html`):** убрана верхняя полоса `cover-bg` — она всегда рисовала яркий градиент, даже когда уже есть фото; теперь один спокойный блок: аватар/инициал + текст, без «лишнего» баннера.
- **Люди / превью профиля / библиотека / лента / сегодня:** вместо цветных градиентных обложек — нейтральные `bg-ink-700` / `bg-ink-800` и тонкая обводка; лента мероприятий — компактная шапка с бейджами без высокого градиента.
- **Антикеш и Safari:** заголовки `no-cache`, `Pragma`, `Expires: 0` для страниц с `_no_store_headers`; на `/feed` и `/today` при восстановлении вкладки из bfcache (`pageshow` + `persisted`) — полная перезагрузка, чтобы лента не залипала на старом HTML.
- **Лента:** у карточек снова есть `id="e-{{ e.id }}"` и `scroll-mt-24` для переходов с «Сегодня».
- **`web/app.py`:** удалены неиспользуемые фильтры `event_cover` и `avatar_cls` и их хелперы.

## Зачем

Меньше визуального шума; страница человека больше не «ломается» визуально из‑за обязательного градиента сверху. Мероприятия чаще совпадают с тем, что в БД, особенно в Safari.

## Почему так

Перезагрузка только на bfcache — компромисс: не опрашиваем сервер на каждом фокусе, но убираем типичный залипший снимок страницы при «назад» в браузере.

## Что улучшить позже

Если залипание всплывёт на других экранах — вынести общий фрагмент `extra_scripts` или добавить лёгкий `fetch` только для счётчика/списка без полного reload.
