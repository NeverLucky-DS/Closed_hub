# change_34

## Что добавлено

- В **библиотеке** у выбранного файла (если ты владелец): форма «Закрепить за компанией» — выбор компании и привязка. На странице **файлов компании** форма привязки убрана, только список и ссылка в библиотеку.
- **Удаление компании**: кнопка на хабе (автор карточки или веб-админ), с подтверждением; папка `media/company/{id}` удаляется вместе с записями в БД.
- **Собеседование в боте**: перед сохранением Mistral отдаёт краткое превью (компания, позиция, дата и т.д.) и кнопки «Сохранить» / «Исправить». Поток `interview_confirm`, колбэки `ivok` / `ived`.
- **Формат записи в файл**: при голосе — строка `Кратко: …` + сырой текст; без голоса — только текст пользователя. Разделитель между записями `---`.

## Зачем

- Закрепление файла логичнее из контекста файла, а не из карточки компании.
- Нужен способ убрать ошибочную компанию.
- Пользователь должен видеть, что бот понял, прежде чем писать в файл и на сайт.
- Старый формат с длинными блоками был нечитаемым.

## Почему так

- Превью через отдельный промпт `prompts/interview_preview_user.txt` и функцию `interview_confirmation_preview` в `services/llm.py`.
- Коммит вынесен в `commit_interview_pending`, превью — в `build_interview_pending` / сессия `interview_pending`.
- Удаление компании: каскад в БД + `shutil.rmtree` для медиа.

## Что улучшить позже

- i18n для превью / кнопок.
- Единый CSRF для POST форм на сайте.
