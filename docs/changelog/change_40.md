# change_40

## Проблема

Ссылки «открыть PDF» / «скачать» с `target="_blank"` на iPhone уводили в системный просмотр (Quick Look, «1 из 5») **вне интерфейса хаба**, без кнопки «Назад» в PWA.

## Решение

- **Просмотр PDF на телефоне:** страницы рендерятся **внутри сайта** через **PDF.js** (cdnjs 3.11.174), `getDocument({ url, withCredentials: true })` к тому же `/api/library/.../raw`, что и раньше. Пользователь листает канвасы внизу, шапка хаба и «К списку файлов» остаются в привычном потоке.
- **Скачивание на телефоне:** кнопки «Сохранить…» — это **`fetch(dl_url) → blob → временный object URL → <a download>`** в **той же вкладке**, без перехода на raw-URL в Safari. На десктопе по-прежнему обычная ссылка «Скачать».
- **Картинки:** убрана ссылка «открыть в новой вкладке»; превью в ленте + `touch-action` для жестов.

## Зависимости

- Нужен доступ к cdnjs с клиента. При жёстком CSP — добавить хост или вынести pdf.js локально в `/static`.

## Ограничение

- На части версий iOS система всё равно может показать промежуточное превью после скачивания — это уже поведение ОС; зато просмотр по умолчанию не покидает хаб.
