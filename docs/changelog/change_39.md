# change_39

## Что сделано

- **Мобильная библиотека (файл открыт):** сверху просмотрщика — липкая панель «К списку файлов» (ведёт на `/library?cat=…#library-file-list`, снимает `file=` из URL), с коротким пояснением про отсутствие системной «Назад» в PWA.
- **PDF:** на экранах `< md` встроенный **iframe отключён**; вместо него текст + две кнопки: «Сохранить PDF» и «Открыть для просмотра», обе с `target="_blank"` (на iOS часто выносит в Safari / системный просмотр, откуда проще «Поделиться» / «В файлы»).
- **Скачать:** на мобилке основная кнопка переименована в «Сохранить файл», с `target="_blank"`; на десктопе поведение прежнее (та же вкладка).
- **Картинки:** ссылка «Открыть изображение на весь экран» в новой вкладке; для `.library-preview-img` на узком экране снят `max-height`, чтобы не резать высокие изображения.

## Зачем

- В standalone / «Добавить на экран Домой» встроенный PDF и скачивание в той же вкладке ведут себя непредсказуемо; отдельная вкладка и отказ от iframe на телефоне снимают типичные затыки.
