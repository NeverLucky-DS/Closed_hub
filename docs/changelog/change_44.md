## Что сделано

- Ссылки с названием файла на `/companies/.../files` ведут на `/library?file=<id>` (без `target=_blank` и без прямого `/api/.../raw`).
- Для `GET /library` без `cat`, но с `file`: если файл в статусе `confirmed` и есть `confirmed_category`, ответ — редирект на `/library?cat=<slug>&file=<id>`.

## Зачем

С компании на телефоне открывался сырой PDF в новой вкладке/просмотрщике ОС — ломалась PWA-вёрстка и не использовался мобильный просмотр из библиотеки (PDF.js, sticky, сохранение).

## Почему так

Один вход в просмотр — та же страница библиотеки, что и при выборе файла из темы. Редирект по `file` убирает дублирование `cat` в шаблоне компании и подстрахует старые закладки.
