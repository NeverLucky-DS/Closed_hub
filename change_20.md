# change_20

## Что сделано

- **Шаблон профиля:** подпись лимита резюме использует `hub_settings.web_max_resume_mb` из Jinja globals вместо несуществующего `config` (Flask-стиль), чтобы не было ошибки рендера и значение совпадало с `config.py`.
- **Контекст:** в `web/app.py` добавлен `_templates.env.globals["hub_settings"] = get_settings()` для доступа к настройкам в шаблонах.

## Зачем

Без глобала `config` в Jinja поле «Резюме» могло ломать страницу или показывать пустой лимит.

## Почему так

Один раз при загрузке модуля — достаточно для read-only настроек; не нужно пробрасывать в каждый `TemplateResponse`.

## Что улучшить позже

Если понадобятся динамические настройки без рестарта — читать из БД/конфига в хендлере и передавать в контекст точечно.
