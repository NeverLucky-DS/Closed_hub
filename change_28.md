## Что сделано

- **Папки с кириллическим названием:** `slugify_folder` больше не выкидывает не-ASCII в пустую строку и не падает в `custom`. Используется Unicode-буквы (`\w`), иначе стабильный `cat_<hash>` от исходной строки. Папки на диске могут называться по-русски (как ввод пользователя в нормализованном виде).
- **Бот:** ответ после «своей папки» — человекочитаемое название и отдельная строка «технический ярлык», без вводящего в заблуждение «код custom».
- **Веб-админы:** переменная `WEB_ADMIN_TELEGRAM_IDS` (список Telegram user id через запятую). В `.env.example` указан пример `1202549697`; в проде задать в своём `.env`.
- **Удаление файла из библиотеки:** `DELETE /api/library/{id}` — помечает файл `deleted`, удаляет файл с диска под `storage/library`, только для админов. Кнопка в UI на странице файла.
- **Скрытие новости:** `DELETE /api/events/{id}` — статус `hidden`, попытка `deleteMessage` в группе из `TELEGRAM_GROUP_CHAT_ID`, если есть `published_message_id`. Кнопка в ленте только у админов.
- **Социальный слой в ленте:** таблицы `event_reactions`, `event_comments`; реакции (фиксированный набор эмодзи), комментарии, ссылки на профиль и `tg://user?id=…` у автора новости и авторов комментариев; удаление своего комментария или админом.
- **Люди / профиль:** кнопки перехода в Telegram со списка людей и со страницы профиля.

## Зачем

- Исправить реальный баг с категорией `custom` при русских названиях.
- Дать админу контроль контента на сайте без правок БД.
- Слегка «оживить» хаб: обратная связь по новостям и быстрый контакт в TG.

## Почему так

- Unicode-slug проще транслитерации и не теряет смысл папки; hash — запасной путь для «всё вырезалось».
- Скрытие события (`hidden`), а не `DELETE`, сохраняет историю и FK.
- Реакции с whitelist эмодзи — меньше мусора и злоупотреблений.
