# change_36

## Что сделано

- Отзыв с сайта (`POST /companies/{slug}/review`) после успешной записи в `company_interview_reviews` **дублируется в тот же `.md`**, что использует бот в разделе «Читать собесы» (`services/interviews_store.py`).
- Имя файла считается через **`interview_company_slug(название компании)`** — как при сохранении из Telegram, а не через латинский `companies.slug`, иначе Яндекс оказывался бы в `yandex.md` в БД и в `яндекс.md` у бота.

## Зачем

- В Telegram отдавался только markdown на диске; записи только из веба в БД не попадали в выгрузку.

## Ограничение

- Уже отправленные **до фикса** отзывы с сайта в `.md` не появятся сами — только новые. При необходимости их можно один раз скопировать в файл или кратко продублировать отзыв.
