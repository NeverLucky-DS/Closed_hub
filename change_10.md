# change_10

## Что сделано

- В `bot/main.py` добавлен **`add_error_handler`**: для `telegram.error.Conflict` (HTTP 409) — одно предупреждение в лог без полного traceback; для остальных ошибок — `log.error(..., exc_info=...)`.

## Зачем

- При двух одновременных `getUpdates` лог засорялся стеком; по смыслу это операционная проблема (два экземпляра бота), а не падение кода.
