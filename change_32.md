## Что сделано

- **БД (миграция 11):** таблицы `companies`, `company_files`, `company_interview_reviews`; у `hr_contacts` поле `company_id` (FK, ON DELETE SET NULL). Ограничение на отзыв: ≥10 символов текста или указан `hr_contact_id`.
- **Хранилище:** `company_root()` → `storage/companies/{id}/…`, раздача `/media/company/{id}/{file}` по whitelist путей в `photo_paths`.
- **Сайт:** раздел «Компании» в навигации; `/companies` (сетка с превью и счётчиками), `/companies/new` (название, описание, до 5 фото — валидация до INSERT в БД), `/companies/{slug}` (HR, файлы по id из библиотеки, отзывы; привязка/отвязка HR; без переноса HR между компаниями без отвязки).
- **Утилита:** `utils/company_slug.py` — slug из названия (кириллица → латиница).

## Зачем

Единая карточка компании для контактов HR, материалов из файловой системы (через записи `files`) и отзывов о собесах.
